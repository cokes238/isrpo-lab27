## Краткое описание работы

В ходе лабораторной работы был создан первый веб-сервер на ASP.NET Core с использованием Minimal API. Изучены основы HTTP-протокола, маршрутизация, параметры маршрутов, возврат JSON-данных, работа middleware и принципы построения веб-приложений на C#. Реализован мини-API сервер с маршрутами для приветствия, информации о студенте, калькулятора и обработки ошибок 404.

## Итоговая таблица: ASP.NET Core Minimal API

| Характеристика | ASP.NET Core |
|----------------|--------------|
| Создание сервера | `WebApplication.CreateBuilder(args)` |
| Сборка приложения | `builder.Build()` |
| Запуск | `app.Run()` |
| Маршрут GET | `app.MapGet("/", fn)` |
| Параметр маршрута | `{name}` → `(string name)` |
| Возврат JSON | `return new {...}` / `Results.Ok(...)` |
| Middleware | `app.Use(async (ctx, next) => ...)` |
| Логирование | Встроенное + `Console.WriteLine` |
| Статус ответа | `Results.NotFound(...)`, `Results.Ok(...)` |
| Тип данных | Строгие (C# `record`, `class`) |

## Ответы на вопросы

### 1. Чем Minimal API отличается от MVC в ASP.NET Core?

| Аспект | Minimal API | MVC (Controller-based) |
|--------|-------------|------------------------|
| Синтаксис | Лямбда-функции прямо в `Program.cs` | Отдельные классы-контроллеры |
| Структура | Всё в одном файле | Логика разбита по файлам |
| Подходит для | Маленькие проекты, микросервисы, прототипы | Крупные проекты с множеством маршрутов |
| Дополнительные возможности | Минимум магии | Фильтры, модели привязки, валидация атрибутами |
| Сложность для новичка | Низкая (порог входа ниже) | Средняя |

### 2. Что произойдёт, если поставить `app.MapGet(...)` до `app.Use(...)`?

Middleware, зарегистрированный после маршрутов, не будет обрабатывать запросы к этим маршрутам. Запрос дойдёт до маршрута раньше, и middleware просто не выполнится. **Порядок регистрации критически важен** — middleware должны быть объявлены до маршрутов, чтобы перехватывать запросы до того, как они попадут в обработчик.

### 3. Почему ASP.NET Core преобразует PascalCase в camelCase при сериализации JSON?

Это сделано для соответствия стандартам JavaScript/JSON, где принято использовать `camelCase` для имён свойств (например, `firstName`, а не `FirstName`). В C# принят `PascalCase` для публичных свойств. ASP.NET Core использует `System.Text.Json` по умолчанию, который применяет `camelCase` для совместимости с клиентским кодом на JavaScript. Это поведение можно настроить через опции сериализации.

### 4. Что означают коды ответа HTTP?

| Код | Название | Значение |
|-----|----------|----------|
| 200 | OK | Запрос успешно выполнен, данные возвращены |
| 401 | Unauthorized | Не авторизован — требуется аутентификация (например, ключ API) |
| 404 | Not Found | Ресурс по указанному URL не найден |

### 5. Чем `dotnet run` отличается от `dotnet watch run`?

| Команда | Поведение |
|---------|-----------|
| `dotnet run` | Запускает сервер один раз. При изменении кода нужно остановить (Ctrl+C) и запустить заново. |
| `dotnet watch run` | Запускает сервер в режиме наблюдения. При любом изменении файлов `.cs` автоматически перезапускает сервер. Это ускоряет разработку. |

## Главные выводы

1. **ASP.NET Core — мощный и производительный фреймворк**, который работает на всех платформах (Windows, macOS, Linux). Он входит в топ самых быстрых веб-фреймворков в мире.

2. **Middleware — это цепочка обработки запросов.** Каждый запрос проходит через все зарегистрированные middleware до того, как попасть в маршрут. Это позволяет легко добавлять логирование, аутентификацию, сжатие ответов и другие сквозные функции.

3. **Маршрутизация в ASP.NET Core интуитивно понятна:** параметры извлекаются автоматически по типу (`int`, `string`, `double`), а возврат объектов C# автоматически превращается в JSON без дополнительного кода.

4. **Minimal API — отличный старт для начинающих.** Он позволяет быстро создать веб-сервер с минимальным количеством кода, сохраняя при этом всю мощь ASP.NET Core. Понимание Minimal API облегчает переход к Controller-based API в будущем.

5. **Правильные HTTP-статусы и обработка ошибок** (например, маршрут `MapFallback` для 404) — это основа качественного API, который легко использовать из клиентского кода (JavaScript, мобильные приложения).
